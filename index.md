You put all that time and effort into bulding all of your courses in Moodle LMS when your school decided to migrate into Canvas LMS. Now you have to start _all over_ and remake everything in Canvas, right?
<br><br>
_Wrong!_
<br><br>
This guide was created to ensure that you _minimize_ the amount of work you need to do in order to transition your course from Moodle into Canvas! This way, you will both save time and effort _not_ starting over from scratch.
<br><br>
For this follow-along guide, I will be migrating my **MAT-263 Brief Calculus** course from Moodle into Canvas.
<br><br>

---

## Before You Export {#before-export}

Before we jump into the "How To" of all this, let's understand the big difference between how Moodle works compared to how Canvas works. More specifically, how does each LMS organize their content?
<br><br>
Moodle is very straight-forward: everything is located on a single page, so if you want to find anything, _just scroll down_. Here is a picture of the top of my Moodle course:
<br><br>
![Moodle course homepage](moodle-to-canvas/assets/images/main_course_page.png "Moodle homepage")
<br><br>
To find the course content, just scroll down.
<br><br>
![Moodle course content](/assets/images/main_course_sections.png "Moodle course content")
<br><br>
Canvas does not have everything located in a single page. Instead, the user starts off at the _Homepage_. Here is a picture of my Canvas course homepage:
<br><br>
![Canvas course homepage](/assets/images/canvas_homepage.png "Canvas homepage")
<br><br>
In order to access the course content, you need to click on a _Navigation link_ on the left called _Modules_.
<br><br>
![Navigation links](/assets/images/modules_link.png "Navigation links")
<br><br>
This will take you to the course content:
<br><br>
![Canvas course content](/assets/images/main_course_sections_Canvas.png "Canvas course content")
<br><br>
Notice that the content area of both Moodle and Canvas look similar. Our goal is to set up our Moodle course in such a way so that when we _do_ migrate  it into Canvas, most of the work has already been done!
<br><br>
To understand how we should organize our course content in Moodle, we first need to know what types of objects are in Moodle, and then we need to know what types of objects are in Canvas. Once we know what objects are which, we can see how they translate from Moodle into Canvas.
<br><br>
Let's start with the biggest and most import objects in both Moodle and Canvas: _Sections_ and _Modules_. If you haven't already noticed, if it is called a _Section_ in Moodle, it is called a _Module_ in Canvas. This means that when you download your course in Moodle in order to migrate it into Canvas, all the Sections in your Moodle course will become Modules in your Canvas course (they will even have the same title). This is convenient because it means the organization of most of your Moodle course will remain the same.
<br><br>
From here, it's just a matter of making sure that all of the objects inside your Moodle sections are set up to become the correct object in Canvas. To see how this works, let's look at the first few sections in my Brief Calculus Moodle course. We start by expanding the _START HERE_ section at the top of my course Moodle page.
<br><br>
![START HERE section](/assets/images/start_here_section.png "START HERE section")
<br><br>
It looks like it contains 3 objects, but it actually contains 4 objects:
1. a text area
2. a quiz
3. a file
4. a link to an external website
<br><br>
The first object is the _text area_. This is the large box that says **COMPLETE THE FOLLOWING BELOW.** This will become its own object in Canvas known as a _Page_. A Page in Canvas is just that: it's own page that takes up the entire screen. You cannot see anything below a page in the same way you can see below a text area in Moodle. Therefore, a page is not as convenient as a text section in Moodle, so you will end up using these objects differently. In order to use a Page properly, it will have EVERYTHING it needs so that it can be self-contained. In Moodle, the student sees a text area and then scrolls down (this can be great for instructions). In Canvas, this would be its own entire page, so the student would then need to navigate somewhere else after reading the text. In essence, _a text area in Moodle by itself is useless in Canvas. Therefore, if you want to use some text, you will need to structure a page so that it works standalone by itself._
<br><br>
The next object is a _Quiz_. This is labeled **1. Click HERE to complete the Enrollment Verification Quiz.** Luckily, Canvas also has its own internal quiz structure. In fact, Canvas has _TWO_ kinds of quiz structures: _Classic Quiz_ and _New Quiz_. Quizzes will typically transfer as a classic quiz, but the results can be discombobulating. I suggest you create a brand new quiz in Canvas and copy and paste questions from one into the other. In addition to this, I suggest you only work in _New Quizzes_ in Canvas, as Classic Quizzes is being phased out. Once Classic Quizzes is gone, so are all those assignments.
<br><br>
The next object is a _file_. This is labeled **2. Click HERE to read the syllabus.** Canvas has a great file structure. You can upload files into a repository for that course, then you can link to that file from anywhere in the course (for example, you can link to that file from inside a Page). Then, when you copy your Canvas course over into the next semester, all the files _AND_ all the links will copy over into the new course. And you won't have to fix anything.
<br><br>
The next object is an External Link. This is labeled **3. Click HERE to sign up for Hawkes Learning.** This is just a link to a URL that is outside of Canvas.
